Section 4.5 The Small World Model

Today, we will talk about (a variation of) the Small World Model of Strogatz and Watts. Their goal was to create a network model that has high clustering (lots of triangles) and small diameter (short connecting paths).

Exercises Practice Problems

1. Clustering Coefficient for a Ring World.

You will calculate the clustering coefficient

\begin{equation*} C = \frac{3 \times (\mbox{# of triangles})}{\mbox{# of connected triples}} \end{equation*}

where a "connected triple" are three vertices \(u,v,w\) such that \((u,v)\) and \((v,w)\) are edges. Intuitively, the clustering coefficient measures the fraction of times that "pairs of my friends are also friends."

  1. Show that there are

    \begin{equation*} n {c/2 \choose 2} \end{equation*}

    triangles in a Ring World. A nice way to argue this is to recognize that all triangles look like one "long edge" going distance \(\leq c/2\) and two "shorter edges." Orient the long edge clockwise the two shorter edges counter clockwise. Then associate this directed cycle with the vertex at the start of the long clockwise arc. Solution.

    We must pick a vertex and then pick two of its right-neighbors.

  2. Show that there are

    \begin{equation*} n {c \choose 2} \end{equation*}

    connected triples. These are paths of length 2 (which can be associated with their central vertex). Solution.

    We must pick the central vertex, then pick two of its neighbors.

  3. Show that the clustering coefficient is

    \begin{equation*} C = \frac{3(c-2)}{4(c-1)}. \end{equation*}

    Furthermore, for \(c \geq 3\text{,}\) we have \(\frac{3}{8} \leq C \leq \frac{3}{4}\text{.}\) This is independent of the network size \(n\text{,}\) so we have high clustering. Solution.

    The clustering coefficient is

    \begin{equation*} \frac{3 n {c/2 \choose 2}}{n {c \choose 2}} = \frac{ 3 (c/2) (c/2-1)}{c(c-1)} = \frac{3(c-2)}{4(c-1)}. \end{equation*}

2. Distances in a Ring World.

  1. Consider vertices \(v_0\) and \(v_{m}\) where \(0 \leq m \leq n/2\text{.}\) What is the graph distance between these nodes? Solution.

    The graph distance is

    \begin{equation*} d(v_0, v_m) = \left\lceil \frac{m}{c/2} \right\rceil = \left\lceil \frac{2m}{c} \right\rceil. \end{equation*}

  2. What is the length of the "longest shortest path" starting at \(v_0\text{?}\) Solution.

    The longest shortest path goes to vertex \(v_{n/2\)} with distance

    \begin{equation*} d(v_0, v_{n/2}) = \left\lceil \frac{n/2}{c/2} \right\rceil = \left\lceil \frac{n}{c} \right\rceil. \end{equation*}

  3. Show that the average distance from \(v_0\) to other vertices is less than or equal to

    \begin{equation*} \frac{n}{2c} + 1. \end{equation*}

    Hint: you can use the upper bound \(\lceil x \rceil \leq x + 1\) for any \(x \in \R\text{.}\)

Solution.

\begin{align*} \frac{2}{n+1} \sum_{m=1}^{n/2} \left\lceil \frac{2m}{c} \right\rceil &\leq \frac{2}{n+1} \sum_{m=1}^{n/2} \left( \frac{2m}{c} + 1\right) = \frac{2}{n+1} \left( \frac{2}{c} \frac{(n/2) (n/2+1)}{2} + \frac{n}{2} \right)\\ &= \frac{2}{n+1} \left( \frac{n(n+2)}{4c} + \frac{n}{2} \right) \leq \frac{n}{2c} + 1 \end{align*}

3. Clustering Coefficient of a Small World.

We turn the Ring World into a Small World by adding some random edges to the graph. We call these edges "shortcuts" because they traverse across the middle of the ring.

For each existing edge in the Ring World, we add a random edge with constant probability \(p\text{.}\) So we add a total of

\begin{equation*} p\frac{cn}{2} = \frac{1}{2} cpn \end{equation*}

random edges to the graph.

  1. Define

    \begin{equation*} q = \frac{\frac{cpn}{2}}{{n \choose 2}} \approx \frac{cp}{n}. \end{equation*}

    For any potential shortcut edge \(e\text{,}\) explain why

    \begin{equation*} \P[ \mbox{edge } e \mbox{ is added to the graph}] \leq q. \end{equation*}

    Solution.

    We add a total of \(\frac{cpn}{2}\) edges out of a possible \({n \choose 2}\) edges. This fraction is an upper bound because we could choose the same edge twice.

  2. For a fixed vertex \(v_i\text{,}\) explain why

    \begin{equation*} \E[ \mbox{# shortcut edges incident with } v_i] = cp. \end{equation*}

    Solution.

    We add a total of \(\frac{cpn}{2}\) edges. Each time we add an edge, the probability that \(v_i\) is one of the endpoints is \(2/n\text{.}\)

  3. Next, we show that we have only added a constant number of new triangles. Show that

    \begin{equation*} \E[\mbox{# triangles using one shortcut edge} ] \leq n{ c \choose 2} q \approx pc^3/2. \end{equation*}

    So we are adding a constant number of triangles. For large \(n\text{,}\) these new triangles are negligible compared to the \(n { c/2 \choose 2}\) triangles already present in the ring.

    Solution.

    Such a triangle has two ring edges. We have \(n\) choices for the central vertex. We then have \({c \choose 2\) ways to choose its two neighbors. Finally, we add the shortcut edge with probabilty \(q\text{.}\)

  4. Triangles using 2 or 3 shortcut edges are also unlikely. Explain why

    \begin{equation*} \E[\mbox{# triangles using two shortcut edges} ]= n \frac{c}{2} (n-2) q^2 \approx \frac{c^3p^2}{2}. \end{equation*}

    and

    \begin{equation*} \E[\mbox{# triangles using three shortcut edges} ] = {n \choose 3} q^3 \approx \frac{c^3p^3}{6}. \end{equation*}

    So we only add a constant number of triangles of any kind.

    Solution.

    A triangle with two shortcut edges contains one ring edge (which we orient clockwise so that we only count it once) and a third vertex. We have \(n\) choices for the first endpoint of the ring edge. We have \(c/2\) choices for the other endpoint. Then we have \(n-2\) choices for the third vertex. We multiply by \(q^2\) which is the probability that both shortcut edges are present.

    As for a triangle with three shortcut edges, we cn choose the vertices in \({n \choose 3}\text{.}\) We multiply by \(q^3\) which is the probability that all three shortcut edges are present.

  5. Turning to connected triples, show that

    \begin{equation*} \E[ \mbox{# connected triples using one shortcut edge} ] = \frac{cpn}{2} (c + c) = c^2 p n. \end{equation*}

    and that

    \begin{equation*} \E[ \mbox{# connected triples using two shortcut edges} ] = n {n-1 \choose 2} q^2 \approx \frac{c^2 p^2}{2} n. \end{equation*}

    Solution.

    For each shortcut edge, there are \(c\) ring edges incident with each enpoint. So the expected number of such triples is \(\frac{cpn}{2} (c + c) = c^2 p n\)

    For each pair of shortcut edges incident with the same vertex, we must choose the other endpoints. So there are \(n\) ways to choose the common vertex, and \({n-1 \choose 2}\) ways to choose the other endpoints. We multiply by \(q^2\) since this is the probability that both shortcut edges are present.

  6. We are now ready to approximate the clustering coefficient. By parts (c) and (d), we can ignore the new triangles (there are only a constant number of them). But we need the new connected triples from part (e). The clustering coefficient is

    \begin{equation*} C \approx \frac { 3 n {c/2 \choose 2} } { n {c \choose 2} + c^2pn + \frac{1}{2} c^2p^2 n } = \frac { 3{c/2 \choose 2} } { {c \choose 2} + c^2p + \frac{1}{2} c^2p^2. } \end{equation*}

    This is a constant (independent of \(n\)), so we have a nonzero clustering coefficient for every small world network.

    Use the approximation \({a \choose 2} = a^2/2\) and set \(p=1\) to obtain the rough lower bound

    \begin{equation*} C \gtrapprox \frac{3}{16}. \end{equation*}

    Solution.

    \begin{align*} C & \geq \frac{3 \left( n { c/2 \choose 2} + c p n \cdot {c \choose 1} \right)} {n {c \choose 2} + c^2pn + n{cp \choose 2} } \geq \frac{3 \left( c/2(c/2-1)/2 %+ c^2 p \right)} {c(c-1)/2 + c^2 + {c(c-1)/2}}\\ & = \frac{3 \left( c-2 \right)} {8 (2c-1)} \geq \frac{3 \left( c-2 \right)} {16(c-1)} \approx \frac{3}{16} \end{align*}
