Therefore, each vertex will get two scores instead of one: an authority score and a hub score. Therefore, we will replace the one-step recursive mantra "important vertices have important neighbors" with the following two-step recursive mantra:

  • If \(v\) has a high authority score, then \(v\) has in-neighbors with high hub scores.

  • If \(v\) has a high hub score, then \(v\) has out-neighbor with high authority scores.

The HITS algorithm works as follows

  • Initialize hub score \(\leftarrow\) 1.

  • Repeat:

    • Update the authority score \(\leftarrow\) sum on in-neighbors' (new) hub scores.

    • Update the hub score \(\leftarrow \) sum of out-neighbors' authority scores.

    • Normalize the scores so that each maximum is 1.

Here is what it looks like on a bipartite directed graph where all the edges point from set \(X\) to set \(Y\text{.}\) In this case, the authority scores in \(X\) will always be 0, and the hub scores in \(Y\) will always be 0. So we only show the hub scores for \(X\) and the authority scores for \(Y\text{.}\) We also skip the normalization.

Let's give a mathematical description of this rule. The HITS algorithm's double recursion updates the hub score \(h_i\) and the authority score \(a_i\) for each vertex. The recursion is:

  • Initially, \(h_i(0) = 1\) for every vertex \(i\)

  • Repeat until convergence for \(t \geq 1\text{:}\)

    • For every vertex \({i}\text{,}\) update \(\displaystyle{a_i(t) = \sum_{j \rightarrow i} h_j(t-1)}\)

    • For every vertex \(j\text{,}\) update \(\displaystyle{h_j(t) = \sum_{j \rightarrow i} a_i(t)}\)

    • Normalize both values.
